frp_easy 架构总览

单 Go 二进制 + Vue 3 SPA，通过 Web UI 管理 frpc（客户端）和 frps（服务端）进程的生命周期与配置。

● Go 1.22 ● Vue 3 + TypeScript ● SQLite (纯Go) v0.1.0 MVP

🗂 单文件部署

整个应用（Go 后端 + Vue 前端）打包为一个可执行文件。embed.FS 将编译后的 SPA dist/ 嵌入二进制，无需独立 Web 服务器。

🔒 无 CGO 依赖

使用 modernc.org/sqlite 纯 Go 实现的 SQLite，CGO_ENABLED=0 编译，可交叉编译到 Linux/Windows 无需系统库。

⚙️ 配置即生效

任何配置变更（代理规则、服务端设置）都会立即触发 DB→TOML 管道：读 DB → 渲染 TOML → 原子写入文件 → 通知 frpc/frps 热重载。

核心数据流 配置变更如何生效

从用户点击"保存"到 FRP 进程收到新配置，完整链路如下：

1

用户提交配置

前端发送 PUT /api/v1/proxies/{id}（或 /server、/client）请求，携带 X-CSRF-Token header 和 session cookie。

2

Handler 写入 SQLite

handler 校验参数后调用 store.UpsertProxy() 或 store.KVSet() 写入数据库，乐观锁（version 字段）防并发冲突。

3

触发 DB→TOML 管道

handler 同步调用 applyConfigBestEffort(ctx, "frpc")，进入 renderAndApplyFrpc()：从 KV 读 serverConn + 从 DB 读已启用代理列表。

4

渲染并原子写入 TOML

调用 frpconf.RenderFrpc(input) 生成 TOML 内容，再通过 frpconf.AtomicWrite(path, data) 写入 runtime/frpc.toml（先写 .tmp 再 rename，防写入一半损坏）。

5

通知 frpc 热重载

调用 procmgr.ApplyConfigChange("frpc")，ProcMgr 通过 frpcadmin 调用 frpc admin API POST /api/reload，frpc 无需重启即可加载新配置。

6

前端更新状态

API 返回 200，前端更新本地状态；进程状态由独立 2s 轮询（GET /api/v1/proc/status）保持同步。

模式切换（Dashboard 开关）： PUT /api/v1/mode 除了写 KV 外，还会立即执行 applyModeToProc：先写 TOML，再调用 ProcMgr.Start/Stop，并将 mode.frpc.enabled 持久化到 KV，重启后自动恢复（AC-9）。

启动序列 main.go

①

加载 appconf

读 frp_easy.toml，解析 UIBindAddr / UIPort / DataDir / LogDir。

②

初始化 Storage

storage.Open(dataDir) 打开/创建 SQLite，运行 SQL 迁移。数据库损坏时自动改名 + 重建（AC-12）。

③

初始化服务层

binloc.NewDefault → procmgr.New（注入 ConfigPaths + frpcadmin reloader）→ auth.NewRateLimiter。

④

启动 HTTP 服务器

httpapi.New(deps) 构建 chi router → net.Listen → srv.Serve（ReadyGate 此时还在拦截，返回 503）。

⑤

自动恢复进程（AC-9）

读 KV mode.frpc.enabled / mode.frps.enabled，若为 true 且二进制存在则自动 Start。

⑥

开放 ReadyGate

ready.Store(true)，所有 API 请求正常处理。监听 SIGINT/SIGTERM，收到信号后优雅关停子进程 → HTTP server。

安全设计 多层防护

🔐

argon2id 密码哈希

m=64MiB · t=3 · p=2，PHC 格式存储，暴力破解极难。

🛡️

CSRF 防护

所有写方法验证 X-CSRF-Token header，防跨站请求伪造。

🚦

登录限速

同 IP 5 次失败后 → 429 + Retry-After，存储在 SQLite KV。

🍪

Session Cookie

HttpOnly + SameSite=Strict，12 小时 TTL，服务端校验。

🔑

Token 脱敏

GET 返回的 authToken 替换为 "***"，需 ?reveal=1 才显示原值。

🌐

默认仅本机

UIBindAddr 默认 127.0.0.1，对外暴露时 stderr 打 WARN。

🔒

原子文件写入

TOML 先写 .tmp 再 rename，防止部分写入导致配置损坏。

🔄

DB 损坏自恢复

SQLite 损坏时自动改名备份 + 重建，用户重新 setup 即可。

数据模型 SQLite

三张表：users（账户）、proxies（代理规则）、kv（键值对配置）。所有配置持久化在 KV 表。

KV 表关键 Key

Key类型说明
frpc.serverConnJSONfrpc 连接配置：serverAddr / serverPort / authMethod / authToken
frps.configJSONfrps 配置：bindPort / authToken / dashboard 设置
frpc.adminJSONfrpc admin API 凭据：addr / port / user / pass（自动生成）
mode.frpc.enabledbool string"true"/"false"，控制重启后是否自动启动 frpc（AC-9）
mode.frps.enabledbool string"true"/"false"，控制重启后是否自动启动 frps（AC-9）

构建与部署

生产构建

  • cd web && npm run build → dist/ 输出到 internal/assets/dist/
  • CGO_ENABLED=0 go build ./cmd/frp-easy → 单一可执行文件
  • 文件内嵌：//go:embed all:dist
  • 版本注入：-ldflags "-X main.Version=x.y.z"

开发模式

  • go run ./cmd/frp-easy（后端，端口 8080）
  • cd web && npm run dev（Vite，端口 5173）
  • Vite proxy：/api → http://127.0.0.1:8080
  • scripts/start.sh 或 start.ps1 一键启动
运行依赖： 将 frp 二进制（frpc / frps）放入 frp_win/（Windows）或 frp_linux/（Linux）目录，与 frp_easy 可执行文件同级。数据库文件自动创建在 dataDir（默认 .frp_easy/），无需手动初始化。
