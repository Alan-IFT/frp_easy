## 1. Goal（目标）

为非技术用户提供一个开箱即用的本地 Web 工具，把 FRP（`frpc` / `frps`）的 TOML 配置与进程生命周期管理收敛到浏览器界面，使用户在不手写 TOML 的前提下完成"设置登录密码、增删端口转发规则、启停 FRP 进程"这三件最常用的事。

---

## 2. In-scope behaviors（本期内必须实现的行为）

> 全部以"用户从浏览器访问 `http://127.0.0.1:<UI 端口>`"为入口。每条均为可观察、可测试。

### 2.1 首次启动与管理员密码

1. **B-1**：首次启动后，访问任意非登录页一律 302 重定向到 `/setup`。
2. **B-2**：`/setup` 页面要求用户设置管理员**用户名**（默认填充 `admin`，允许修改）与**密码**（必填，至少 12 个字符，必须含字母与数字；前端校验 + 后端再次校验）。
3. **B-3**：`/setup` 提交成功后，凭据落入持久化存储；密码以加盐哈希形式保存（明文密码不写盘、不写日志）。
4. **B-4**：`/setup` 完成后自动登录并跳到 `/dashboard`；之后再访问 `/setup` 一律 404 或 302 到 `/dashboard`。
5. **B-5**：`/login` 页面接受用户名 + 密码；成功登录后会话有效期默认 12 小时；登录失败 5 次同 IP 触发 60 秒退避。
6. **B-6**：登录态可在「设置」页修改密码；旧密码必须先通过校验。

### 2.2 模式选择（运行 frpc / frps / 同时运行）

7. **B-7**：在 `/dashboard` 用户可独立开启「客户端（frpc）」与「服务端（frps）」两个开关。两者**互不强制**：仅开 frpc、仅开 frps、两者同开均合法。
8. **B-8**：每个开关旁实时显示进程状态：`stopped` / `starting` / `running` / `stopping` / `error`，并显示该进程的当前 PID（若有）和最近一次状态变更时间。
9. **B-9**：模式开关状态跨重启保留（持久化）；本机重启 UI 服务后，先前为「开」的进程在 UI 服务就绪后**自动恢复**为运行中。

### 2.3 端口映射 / 代理规则的增删改查（针对 frpc）

10. **B-10**：`/proxies` 页面展示当前所有代理规则的列表，列：`name` / `type` / `localIP:localPort` / `remotePort 或 customDomains` / `enabled`。
11. **B-11**：用户可「新增」一条规则，必填字段：
    - `name`（字符串，1–64 字符，正则 `^[A-Za-z0-9_-]+$`，全局唯一）
    - `type`（枚举：`tcp` / `udp` / `http` / `https`，本期只支持这四种）
    - `localIP`（默认 `127.0.0.1`）
    - `localPort`（1–65535）
    - 当 `type ∈ {tcp,udp}`：`remotePort`（1–65535）
    - 当 `type ∈ {http,https}`：`customDomains`（数组，至少 1 项；每项为合法域名）
    - `enabled`（默认 `true`）
12. **B-12**：用户可对任一规则执行「编辑」「启用/禁用」「删除」；删除前 UI 必须二次确认。
13. **B-13**：任一新增 / 编辑 / 启停 / 删除提交成功后，UI 必须**立即**让 frpc 加载新配置（通过 frpc 自身的热加载或重启进程实现，对用户透明），并在 5 秒内反馈成功或失败。
14. **B-14**：保存失败时保留用户输入并展示具体字段级错误（端口冲突、name 重复、域名格式错等）。

### 2.4 服务端（frps）最小可配项

15. **B-15**：`/server` 页面允许配置 frps 必备项：
    - `bindPort`（默认 7000，1–65535）
    - `auth.token`（字符串，留空表示不启用 token 鉴权；UI 默认生成一个 32 字符随机串并提示用户复制保存）
    - `dashboard` 开关（开启后 UI 同步配置 `webServer.addr=127.0.0.1`、随机端口、用户名密码与本工具登录态**不复用**，独立保存）
16. **B-16**：客户端（frpc）的服务端连接信息（`serverAddr`、`serverPort`、`auth.token`）作为独立表单项在 `/client` 页面填写；与 `/server` 页面的 frps 配置在数据模型上分离。

### 2.5 进程控制与运行可观察

17. **B-17**：`/dashboard` 提供「启动 / 停止 / 重启」按钮分别作用于 frpc、frps；按钮在执行中禁用，结果在 5 秒内回填。
18. **B-18**：`/logs/frpc` 与 `/logs/frps` 两个页面分别显示对应进程**最近 500 行**日志（尾部）；页面有「自动刷新」开关（默认关），开启时每 2 秒拉取增量。
19. **B-19**：当某进程处于 `error` 时，`/dashboard` 显示醒目错误提示，并展示导致错误的最后 20 行日志摘要 + 「跳转到完整日志」链接。

### 2.6 配置 / 凭据持久化

20. **B-20**：所有用户在 UI 中的输入（管理员凭据、模式开关、frpc/frps 配置、代理规则、会话密钥等）必须跨 UI 服务进程重启保留；持久化文件位于项目工作目录下且**不进入 git**（默认 `.frp_easy/` 目录，由项目 `.gitignore` 排除）。
21. **B-21**：UI 服务启动时若发现持久化数据缺失或损坏（解析失败），必须进入「首次启动」状态并保留损坏文件副本（重命名为 `*.broken-<timestamp>`），不得静默清空。

### 2.7 跨平台与部署

22. **B-22**：UI 服务必须在 Windows 10/11（x64）与主流 Linux x64 发行版（glibc ≥ 2.31）下表现一致；启动脚本同时提供 `.ps1` 与 `.sh` 两种入口。
23. **B-23**：UI 服务启动时自动检测 `frp_win/` 或 `frp_linux/` 中相应的 `frpc` / `frps` 二进制并使用之；若二进制缺失，UI 显示明确指引，不崩溃。
24. **B-24**：UI 服务监听地址默认 `127.0.0.1:<UI 端口>`，**不**默认对外暴露。绑定地址与端口可在配置文件中修改，但 UI 内不提供修改入口（避免误操作锁死自己）。

## 4. Boundary conditions（边界条件）

### 4.1 输入边界

- **空 / null**：所有必填字段提交空 → 422 + 字段级错误，不入库。
- **超长**：`name` > 64 字符、域名 > 253 字符 → 拒绝。
- **非法字符**：`name` 不匹配 `^[A-Za-z0-9_-]+$` → 拒绝。
- **端口范围**：localPort / remotePort / bindPort 必须在 `[1, 65535]`；`< 1024` 在 Linux 下需 root，UI 提交时**警告**但允许保存（执行失败由进程错误回填）。
- **重复**：`proxy.name` 全局唯一；同 `(type, remotePort)` 重复（tcp/udp）→ 拒绝；同 `customDomain` 重复（http/https）→ 拒绝。
- **互斥字段**：`tcp/udp` 不接受 `customDomains`；`http/https` 不接受 `remotePort`。

### 4.2 并发与时序

- 同一资源（例如同一条 proxy 规则）并发编辑：以**最后写入胜出**（last-write-wins），在响应中带上当前版本号供前端检测覆盖。
- 进程「启动」按钮在 starting / running 状态下禁用；「停止」按钮在 stopping / stopped 状态下禁用；防止重复点击导致重复 fork。
- UI 服务启动期间（持久化加载尚未完成）不接受任何写请求，统一返回 503 + 重试提示。

### 4.3 错误路径

- frpc / frps 二进制不存在 → 启动按钮 disable + UI 顶部 banner 提示。
- frpc / frps 启动后 3 秒内自我退出 → 状态置 `error`，捕获 stderr 末尾写入日志页面。
- 持久化目录不可写 → UI 启动失败并以**清晰中文**写出原因到控制台（不静默崩溃）。
- 端口被占用导致 frps 启动失败 → 状态 `error` + 错误日志摘要中包含 `address already in use` 类提示。

### 4.4 资源上限

- 单实例 proxy 规则数：**≤ 200**（超出报错 + 提示拆分）。
- 日志页面单进程历史保留：**≤ 7 天 或 ≤ 100 MB**，先到为准；轮转后老段落不再可在 UI 查看（提供文件路径供用户手动取）。
- 单条字段最大文本：4 KiB（应对极端域名列表 / 注释）。

## 5. Acceptance criteria（验收准则 — 每条可机器/人工验证）

| # | 准则 | 验证方法 |
|---|---|---|
| AC-1 | 全新仓库首次启动后访问 `http://127.0.0.1:<UI 端口>/dashboard` 必返回 302 → `/setup` | `curl -i http://127.0.0.1:<UI 端口>/dashboard` 看 `Location: /setup`，HTTP code 302 |
| AC-2 | `/setup` 提交合法用户名 + 密码（≥12 字符 + 含字母数字）后返回 2xx，且持久化目录下出现凭据文件 | 提交后检查 `.frp_easy/` 下出现凭据文件，且文件中**不包含明文密码**（grep 提交的密码字符串应无命中） |
| AC-3 | 已设置过密码后再访问 `/setup` 返回 302 → `/dashboard` 或 404 | `curl -i .../setup` 检查响应码 |
| AC-4 | 错误密码登录 5 次后第 6 次返回 429（或 403）并保持 60 秒 | 脚本循环 6 次 POST `/api/login` 验证最后一次状态码 ≥ 400 且 `Retry-After` 头存在 |
| AC-5 | 在 `/proxies` 通过 UI 提交一条 `tcp` 规则（name=demo-tcp, localPort=22, remotePort=6000）后，5 秒内 frpc 进程的实际运行配置生效 | UI 列表中出现该条；并通过 frpc 的 admin API（`GET /api/status` 或等价）能查到 `demo-tcp` 在线 |
| AC-6 | 删除该规则后，5 秒内 frpc 不再持有它 | 同 AC-5 反向验证 |
| AC-7 | 启动 frps 后访问 `/dashboard` 看到 `running` 状态 + PID > 0 | UI 显示 + `ps`/`Get-Process` 能找到对应 PID |
| AC-8 | 停止 frps 后 5 秒内 UI 状态变 `stopped` 且 PID 清空 | UI 显示 + 系统进程列表确认已退出 |
| AC-9 | 模式开关「frpc=开、frps=关」状态下重启 UI 服务，UI 就绪后 frpc 自动 `running`、frps 保持 `stopped` | 重启 UI 服务，等待就绪后查 `/dashboard` |
| AC-10 | 提交非法 proxy（重复 name 或 remotePort 冲突）→ HTTP 422，错误体含字段名 | `curl -X POST /api/proxies -d ...` 验证响应 |
| AC-11 | `/logs/frpc` 默认显示 ≤ 500 行；开启「自动刷新」后每 2 秒有一次 GET 请求 | 浏览器 DevTools Network 观察请求频率 |
| AC-12 | 持久化文件被人为破坏（写入 `garbage`）后重启 UI 服务，进入 `/setup` 流程，且原文件被改名为 `*.broken-<timestamp>` | 手工注入损坏 + 重启 + 看目录 |
| AC-13 | 在 `frp_win\` 与 `frp_linux\` 任一缺二进制时，UI 启动**不崩溃**，对应模式开关 disable + banner 给出指引 | 临时改名 frpc / frps 后启动 UI 验证 |
| AC-14 | UI 默认仅监听 `127.0.0.1`；`netstat -an` 看 UI 端口 LISTENING 行的本地地址必为 `127.0.0.1:<UI 端口>` | `netstat -ano` (Win) / `ss -lntp` (Linux) |
| AC-15 | 全栈端到端：从空仓库启动 → 设置密码 → 配置 frpc 连 frps → 添加 tcp 规则 → 通过映射端口建立 TCP 连接成功 | 在同机自连：`ssh -p 6000 user@127.0.0.1`（或等价 nc 测试）能与本地 22 端口握手 |

## 6. Non-functional requirements（非功能需求）

### 6.1 安全

- **NF-S1**：管理员密码必须以加盐哈希落盘（算法可由 Architect 决定，但禁止明文 / Base64 / MD5/SHA1 直存）。
- **NF-S2**：会话凭据（cookie / token）必须 `HttpOnly` + `SameSite=Lax`；UI 走 HTTP 时无需 `Secure`，走 HTTPS 时必须 `Secure`。
- **NF-S3**：所有写接口必须有 CSRF 防护（同源校验或 token，二选一由 Architect 决定）。
- **NF-S4**：默认绑定 `127.0.0.1`；如用户改为 `0.0.0.0`，UI 启动时在控制台打印警告。
- **NF-S5**：FRP 的 `auth.token` 字段在 UI 上以"密码框"展示，需要点击「显示」才以明文出现；导出 / 序列化到响应体时统一脱敏（除非用户主动点击查看）。
- **NF-S6**：日志输出禁止包含明文密码 / token；接入日志前须经过统一脱敏过滤。

### 6.2 可用性 / UX

- **NF-U1**：所有 UI 文案默认中文；按钮、错误提示、表单标签均中文。
- **NF-U2**：每个写操作必须有 loading 状态与成功/失败 toast；表单错误必须**字段级**展示而非顶部一句"出错了"。
- **NF-U3**：所有破坏性操作（删除规则、停止进程）必须二次确认；确认弹窗显式说明影响范围。
- **NF-U4**：键盘可达：所有交互元素可 Tab 聚焦，回车可触发主操作。

### 6.3 性能

- **NF-P1**：UI 列表页（≤ 200 条规则）首屏渲染时间 P95 ≤ 1 秒（本机访问、空缓存）。
- **NF-P2**：日志拖尾接口在 100 MB 日志文件上读取最近 500 行 P95 ≤ 500 ms。
- **NF-P3**：进程启停操作（按下按钮到 UI 显示终态）P95 ≤ 5 秒。

### 6.4 跨平台 / 兼容

- **NF-C1**：UI 在 Chrome ≥ 110、Edge ≥ 110、Firefox ≥ 110 三款主流浏览器下功能与样式一致。
- **NF-C2**：服务端在 Windows 10/11 x64 与 Linux x64（glibc ≥ 2.31）下二进制均可运行。
- **NF-C3**：路径处理跨平台（不写死 `/` 或 `\`）。

### 6.5 可观测 / 可维护

- **NF-O1**：UI 服务自身有结构化日志（级别 ≥ info），写到工作目录下日志文件，按天轮转。
- **NF-O2**：所有对 frpc/frps 子进程的 fork、信号、退出事件必须打日志（含 PID、退出码）。
- **NF-O3**：项目根需要存在 `.env.example`（即便本期不强依赖环境变量，也保留约定）。

## 8. Open questions（开放问题 — 全部 PM 自治模式下自答）

> 每条按 INPUT.md 三原则给出推荐答案并标 `[PM 决策]`，已合并到上面的 In-scope 与 Acceptance。

### Q-1 首次启动是否强制设管理员密码（vs 默认 admin/admin）？

- 候选 A：强制 `/setup` 走一次。
- 候选 B：硬编码 `admin/admin`，登录后提醒改。
- **[PM 决策] 选 A**。依据：原则 ①（默认安全），原则 ②（默认值即弱口令是常见反模式）。已落到 B-1 ~ B-5 与 AC-1 ~ AC-3。

### Q-2 是否同时支持 frpc 与 frps（同机同时运行）？

- 候选 A：仅 frpc 客户端。
- 候选 B：仅 frps 服务端。
- 候选 C：两者独立开关，可单可双。
- **[PM 决策] 选 C**。依据：原则 ①（用户场景覆盖最广，开发机上做穿透 / 服务器做接入两种典型用法都需要）；原则 ②（两个进程模型独立，分别管理是更清晰的架构）。已落到 B-7 ~ B-9。

### Q-3 frpc 配置变更下发方式：热加载 vs 重启进程？

- 候选 A：热加载（依赖 frpc 自带 admin API）。
- 候选 B：直接重启进程。
- 候选 C：先尝试热加载，失败回退到重启。
- **[PM 决策] 选 C**。依据：原则 ①（热加载用户感知最小）；原则 ③（实现细节由 Architect 决定，但本期需求侧只关注"5 秒内生效"，已落到 AC-5 / AC-6）。

### Q-4 持久化存储介质（文件 vs 嵌入式 DB）？

- 候选 A：单一 JSON / TOML 文件。
- 候选 B：嵌入式数据库。
- 候选 C：多文件分目录。
- **[PM 决策] 不在本阶段决定**。依据：技术选型属 Architect 范畴。需求侧只规约「跨重启持久化、损坏需保留副本、不入 git」，见 B-20 / B-21 / AC-12。

### Q-5 UI 服务对外暴露（绑定 0.0.0.0）默认是否允许？

- 候选 A：默认仅 127.0.0.1。
- 候选 B：默认 0.0.0.0。
